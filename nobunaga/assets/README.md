# Assets — the registry

> The single home for every rendered/extracted asset, with a **predetermined structure** so new
> assets land in one place and we stop duplicating. Before saving a render, find its category here;
> if it doesn't fit, add a category (don't dump it flat). Replaces the old flat `atlas/` +
> `commands/assets/` dirs (reorg 2026-06-11).

## Structure

| Dir | Holds | Generated by | Naming |
|---|---|---|---|
| `animations/` | the 21 lord-command + cutscene animations (GIFs) — **canonical, command-named** | `run-animation.py` → promoted | `<command>.gif` (e.g. `grow.gif`); cutscene-only ids as `anim-<id>.gif` |
| `animations/frames/` | per-animation frame-sheet PNGs (debug/reference, build artifacts) | `run-animation.py` | `<command>-frames.png` / `anim-<id>-frames.png` |
| `maps/fiefs/` | the 17 per-fief tactical maps (sram/ppu/full-map renders + the `fief-NN-name` grid set) | `render-atlas.py`, `render-fief-from-ppu.py`, `render-from-sram.py` | `fief-NN-name.png`, `<fief>-{ppu,sram}-render.png` |
| `maps/strategic/` | world / strategic-overview maps + the overview atlases | `render-strategic-map.py`, `render-strategic-atlas.py`, `render-atlas.py` | `strategic-*.png`, `atlas-overview*.png` |
| `portraits/` | daimyo portraits (anthologies, composites) | `render-portrait.py` | `daimyo-*.png`, `portrait-*.png` |
| `screens/` | title / boot / UI / font / turn-loop renders | `render-text.py`, misc | `title*.png`, `text.png`, `font-*.png` |
| `rom/` | raw ROM asset extractions (per-scenario terrain etc.) | `render-rom-atlas.py` | `rom-NN/`, `rom-*.png` |

## Rules (anti-duplication)

- **One canonical copy per asset.** Command animations live ONCE here as `animations/<command>.gif`;
  `build-command-page.py` reads from there and base64-embeds into the page (the page is the only
  "second copy", by design — a single portable file). Do not also stage a numeric `anim-<id>.gif`.
- **Tools write here, not to a flat dir.** Every render tool's output path points into this tree
  (updated 2026-06-11). If you add a render tool, point it at the right category.
- **Promote, don't duplicate.** `run-animation.py` emits `animations/anim-<id>.*` (raw); a command
  animation is *renamed* to `<command>.gif` (canonical), not copied alongside the numeric one.
- The old `atlas/` (started as fief/world maps, became a dumping ground) and `commands/assets/`
  (duplicated the command GIFs) are retired — everything moved here, dedup'd.
