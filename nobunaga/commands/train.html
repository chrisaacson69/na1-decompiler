How it works

The driver $A637 reads as a step-by-step script:

  1. $A63CSelect Train, then the province.
  2. $A647Skill-ceiling guard — proceeds while skill < header (effectively always).
  3. $A655Soldier check. If the province has no men → "You have no soldiers!" — you can’t drill an empty garrison.
  4. $A671The animation plays first — ui_helper_e80c(10) hands id 10 to the bank-14 animation VM (note: Train animates before applying, unlike Grow/Build).
  5. $A676effect_train rolls the skill gain (formula below).
  6. $A679skill is clamped to its ceiling, then the new value is shown.

The formula

Read directly from $9586 bytecode and execution-verified.

// skill drill — pure RNG stat-up, NO cost, NO drain   (effect_train $9586)
skill_gain = (rng%20 + 10) × 4              // 40 .. 116   ((roll+10) << 2)

// aptitude bonus — a gifted daimyo drills his men harder
if daimyo[+3] + daimyo[+5] > rng%10 + 90:    // (Luck + IQ) vs 90..99
    skill_gain += 10                       // → 50 .. 126

skill += skill_gain                          // capped at the fief ceiling

Free skill, but RNG-variable (a session swings 40–116 on the d20 roll). Drill idle garrisons between wars — high skill feeds straight into the combat damage multiplier. There is never a reason not to Train an army that has soldiers and time.

The cleanest command in the game. No gold, no rice, no drains — Train is the only free stat-up, which is exactly why elite armies are trained, not hired (Hire dilutes the average; see hire.html). The price is variance: the d20 roll swings a session 40–116. The aptitude bonus rewards a smart, lucky daimyo — for Tokugawa (aptitude pair = 167, always above the 90–99 threshold) the +10 always fires, so his floor is 50. The 40–80 range in the earlier notes was a misread — the bytecode $959A–$95A2 shifts (roll+10) left twice (×4).

Validated against the ROM

Train is RNG-driven — validated by range, not by point. The d20 roll sets the band; 9/9 in-game Tokugawa sessions landed inside it.
