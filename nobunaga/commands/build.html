How it works

The driver $A853 reads as a step-by-step script:

  1. $A858Select Build from the command menu, then the province.
  2. $A863Limit check. If town ≥ header (development ceiling) → "We’re at our limit." and the command aborts.
  3. $A87CGold check. If the province has no gold → "You have no gold."
  4. $A881Prompt: "Town is N. Spend how much on it?"
  5. $A896Number entry, capped at the province’s current gold.
  6. $A8A2gold −= amount — the driver debits gold before the effect runs.
  7. $A8A9effect_build applies the formula (below).
  8. $A8B0The animation plays — ui_helper_e80c(24) hands id 24 to the bank-14 animation VM.
  9. $A8B6"Town value is now N." → confirm.

The formula

Read directly from $88A6 bytecode and execution-verified.

// gain — inverse-sqrt diminishing returns (Grow’s kernel, on TOWN)
sqrt = int_sqrt(town + amount)
g    = ⌊amount × (6 − const_two) ÷ sqrt⌋       // const_two=$6D63 ≈ 1, so ×5
g    = max(g, 1)
g    = min(g, header − town)             // headroom clamp (rarely fires)

// secondary drain — ONE field (wealth); pct computed LIVE, capped 50%
pct  = (g > town) ? 50 : ⌊100 × g ÷ (g + town)⌋ ÷ 2
wealth −= pct_op(wealth, pct)              // ⌊wealth × pct ÷ 100⌋

// town write — routed through helper_dam_rounding(town_ptr, 2g)
town += 2 × g                             // full add in peacetime; halved if the province is at war
gold −= amount                           // debited by the driver
return g                                // (un-doubled) for the confirm screen

ROI ≈ 10 / √(town + amount) — the identical curve to Grow, but Build feeds gold income (town is in the gold-harvest formula) where Grow feeds rice. Front-load Build while town is low.

Two corrections to the earlier notes (2026-06-02, full bytecode walk + ROM re-execution). (1) The wealth-drain percentage is not the constant 20 — exactly like Grow, the bytecode at $88E4–$88FF computes it live from the gain-to-town ratio, capped at 50%. (2) There is no separate “main + secondary” town add. The whole gain (2g) is added once, through helper_dam_rounding ($887D), which only halves the amount when the province is at war (war_helper_d972), then calls helper_82AC to cap town at its header ceiling. The “secondary bump” in the old draft was a misread of that single routed write.
