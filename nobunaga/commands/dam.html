Dam

How it works

The driver $9B7E reads as a step-by-step script:

  1. $9B83Select Dam, the province. If output == 0 → "You have no peasants!" (you can’t dam undeveloped land).
  2. $9BAALimit check. If dams ≥ 100 → "We’re at our limit." (dams is the only 0–100-capped stat).
  3. $9BBEGold check. No gold → "You have no gold."
  4. $9BCFSpend cap = min(gold, (100 − dams) × √output ÷ 2).
  5. $9BEAPrompt: "Dams is N. Spend how much on it?" → number entry.
  6. $9C08gold −= amount.
  7. $9C0Fgain = ⌊2 × amount ÷ √output⌋ (min 1); dams += gain via helper_dam_rounding (war-halves).
  8. $9C26The animation plays — ui_helper_e80c(21) hands id 21 to the bank-14 animation VM.
  9. $9C2Cdams clamped to 100; "Dams value is now N." → confirm.

The formula

Read directly from $9B7E bytecode and execution-verified.

// preconditions (driver $9B7E)
if output == 0:  // "you have no peasants" — can’t dam undeveloped land
if dams  >= 100: // "we’re at our limit" — dams is the ONLY 0–100-capped stat
if gold  == 0:   // "you have no gold"

sqrt = int_sqrt(output)                       // via $87D8 (the real "effect_dam" is just this)

// spend cap — couples to output AND remaining dam headroom
max_spend = min( gold, (100 − dams) × sqrt ÷ 2 )
amount    = number_input(cap = max_spend)
gold     −= amount

// gain — INVERSELY proportional to √output
gain  = max(1, ⌊ 2 × amount ÷ sqrt ⌋)
dams += gain                            // helper_dam_rounding $887D: war-halves, caps at 100

Dam multiplies with output in the rice harvest (rice += pct_op(output × dams, tax)), so its ROI rises with output even as the gold cost does. Rule of thumb: Dam beats Grow for rice once output² > ~38 × dams — i.e. dam your already-high-output rice fiefs.

The last open formula — and it never lived where the labels said. $87D8 (tagged “effect_dam”) is merely a sqrt(output) helper; the real dam math is computed inline in the driver $9B7E and applied through helper_dam_rounding ($887D, shared with Build). Two defining properties: dams is the only province stat capped at 0–100, and the gain is inversely proportional to √output — a developed fief pays far more gold per dam point. The spend cap (100−dams)×√output÷2 is exactly the amount that lifts dams to 100, so “spend the max” always tops out the dam. Dams matter because the rice harvest multiplies output × dams (appendix §4).
