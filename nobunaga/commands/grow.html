Lord Command · No. 7

Grow

Spend gold to raise a province's agricultural output — on a diminishing-returns curve, at the cost of loyalty and flood-control.

In game

The Grow animation, rendered purely from ROM bytes — no emulator capture.

A peasant works the field while the output meter climbs beside him. FROM ROM

When to use it

ROI ≈ 10 / √(output + amount) — front-load Grow while output is low; returns fall off sharply past output ≈ 300.

Changes: output (+gain) · loyalty (−drain) · dams (−drain) · gold (−amount).
Under the hood — the verified bytecode

How it works

Driver $9D3D reads as a step-by-step script:

  1. $9D4CSelect Grow from the command menu, then the province.
  2. $9D57Limit check. If output ≥ header (development ceiling) → "We’re at our limit." and the command aborts.
  3. $9D68Gold check. If the province has no gold → "You have no gold."
  4. $9D7CPrompt: "Output is N. Spend how much on it?"
  5. $9D88Number entry, capped at the province’s current gold.
  6. $9D99effect_grow applies the formula (below).
  7. $9D9BThe animation plays — ui_helper_e80c(9) hands id 9 to the bank-14 animation VM.
  8. $9DAD"Output is now N." → confirm.

The exact formula

Read straight from $87F0 bytecode and execution-verified.

// gain — inverse-sqrt diminishing returns
sqrt   = int_sqrt(output + amount)
gain   = 2 × ⌊amount × (6 − skill) ÷ sqrt⌋   // skill = const_two $6D63 = the DIFFICULTY level (1-5, default 2)
gain   = max(gain, 1)
gain   = min(gain, header − output)            // headroom clamp (rarely fires)

// secondary drains — pct is computed LIVE, scaling with grow aggressiveness
pct    = (gain/2 > output) ? 50 : ⌊100 × gain ÷ (gain + output)⌋ ÷ 2
loyalty −= pct_op(loyalty, pct)              // ⌊loyalty × pct ÷ 100⌋
dams    −= pct_op(dams,    pct)              // same pct

output += gain
gold   −= amount                            // debited by the driver
The hidden SKILL-LEVEL dial. The (6 − skill) multiplier is the game's difficulty knob (const_two $6D63, chosen 1–5 at “Please select skill level.”, default 2) — and it scales your entire economy. On one province (output 80, spend 100) the gain is 76 / 60 / 46 / 30 / 14 for skill 1 / 2 / 3 / 4 / 5 — a 5.4× swing the UI never shows (emulator + econ_sim verified). Higher skill = slower development = harder. The long-mistaken literal “×5” was just skill-1’s 6−1; the validation table below was captured at skill 1.

Drain correction (2026-06-02). The secondary-drain percentage is not a constant 20 — the bytecode at $8833–$884E computes it live from the gain-to-output ratio, capped at 50%. The “20” was a coincidence of the single drain-measured test (gain 56, output 80 → ⌊5600/136⌋/2 = 20).
