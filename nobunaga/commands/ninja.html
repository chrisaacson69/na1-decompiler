Ninja (Hire ▸ Ninja)

Buy a ninja and send him on a sabotage mission against an enemy fief — uprising, revolt, flood, assassination or arson. The covert half of the Hire command.

What you see

The hired ninja steals out toward the target province (the shared mission-launch animation, id 12) — each mission then plays its own effect animation. FROM ROM

Produced headless by tools/run-animation.py 12, which runs the game’s own animation scripting VM (bank 14 $80AD, id 12 from the descriptor table at $AF80) and intercepts every CHR upload, palette write and sprite placement to reconstruct each frame — real NES palette.

What it does

Reached from the Hire command’s (Men/Ninja)? prompt — pick Ninja and you leave the recruiting sandbox for an offensive espionage subsystem (effect_ninja_sabotage $A2D2). You pay gold to hire the ninja (rate hire_gold_rate $6E13, which re-rolls every turn), choose a target fief (Send where?), then choose one of five missions. A ninja-skill contest gates the outcome: the run proceeds only when your daimyo’s skill stat (record +3) + 30 beats the target daimyo’s — otherwise Your ninja failed! and you forfeit the gold anyway (effect_hire_pay_gold $A29B). On success the target loses the mission’s fields and you see Fief X’s <stat> has declined by N. This is the system the old notes mis-filed under “Bribe” — see the correction below; the genuinely separate Bribe command gets its own page.

How it works

The driver reads as a step-by-step script:

  1. $A615From Hire, answer (Men/Ninja)? with Ninja (ui_helper_d351) → effect_ninja_sabotage $A2D2.
  2. $A2EDPrompt "How many" — number entry (gated by affordability vs hire_gold_rate; no gold → "You have no gold.").
  3. $A31BPrompt "Send where?" — pick the target province.
  4. $A335Prompt "What mission" → the menu "1-Uprisng 2-Revolt 3-Dams 4-Assassin 5-Arson" ($BD8F).
  5. $A36DThe ninja sets out — ui_helper_e80c(12) plays the mission-launch animation (each mission then has its own effect animation; see the table).
  6. $A396Ninja-skill contest. Proceeds only if target_daimyo[+3] < your_daimyo[+3] + 30 — a fixed +30 attacker edge on the skill stat.
  7. $A39AMission switch — dispatch to the chosen sabotage effect (drains via hire_stat_drain_rng, clamped so a field never goes below 0).
  8. $A2ADOn failure → "Your ninja failed!"; gold is spent regardless (effect_hire_pay_gold).

The five missions

Each mission drains specific fields of the TARGET fief and plays its own animation. Read straight from the switch at $A39A.

#MissionDrains on the target fiefAnimCode
1Uprisingloyalty + wealth28$A3AB
2Revoltmorale27$A45A
3Damsdams + rice (flood)30$A497
4Assassinkills the enemy daimyo — gated by “daimyo present” ($6DA2); resolves in ninja_mission_resolve_vs_defender1*$A508
5Arsontown29$A510

Drain magnitude (from hire_stat_drain_rng $A255 bytecode): drain = (rng % max(1, ⌊field × √(your_skill+30) ÷ 100⌋) + 1) × 5 — always a multiple of 5 (max 5×the cap), scaling with the target field’s value and your daimyo’s skill (record +3, with a +30 edge). Each drain is clamped to the field (can zero a stat, never negative). Emulator-confirmed across the full RNG band (tools/probe-espionage.py — the observed drains matched {5,10,…,X×5} for every test vector). *Assassin has no field-drain animation of its own — it runs the resolution sub, which plays animation 1.

This is the “buy → ninja” system — and it is NOT Bribe. The toml long labeled $A2D2 effect_hire, and an earlier note filed the whole Uprising/Revolt/Dams/Assassin/Arson menu under the Bribe command. The bytecode says otherwise: this menu hangs off Hire ▸ Ninja (driver_hire $A5F4), while the recruit-men path is effect_hire_variant_pay $A553 and the real Bribe command ($8D4D) is a separate gold-for-spy peasant-defection. The label is now effect_ninja_sabotage. The sabotage mechanics the old note described were all real — only the command attribution was wrong.
