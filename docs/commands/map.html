Lord Command · No. 18

Map

Open the strategic map. A read-only view — it changes nothing and costs no turn. The one command you can use as often as you like.
① decompiler → C ② data-walk labels ⚡ instant — no animation

What it does

Map (driver_map $AF38) is a pure view. It redraws the strategic map (ui_helper_cd20), branches on scenario size (scenario_fief_count == 50) to choose the 17- or 50-fief layout, looks each fief’s map-id up in battle_init_driver_data_fed8, and paints it via map_helper_e5f2 / map_helper_af10. It writes no game state and returns 0 — i.e. it does not consume your action for the season. There is no animation and no effect handler. The illustrated map is a 9-section pannable picture (arrows change sections, A returns to the menu) blitted from bank 4 — section tilemaps at map_helper_data_8d5c (448 B each), attributes at map_helper_data_9d1c, CHR via ui_helper_cd20. The same render core is reached from the kernel dispatch_map_helper_e694 ($E694) standalone viewer and the View command’s map sub-screen.

Why it’s a one-paragraph page. Map touches nothing — no province field, no daimyo record, no relations matrix — and unlike every other entry on the menu it doesn’t end your turn. Two different map artifacts exist in this project: the schematic node-link adjacency atlases (tools/render-strategic-atlas.py / render-strategic-50.py, built from the adjacency table + fief coordinates) — useful, but not the game’s art — and the actual illustrated map this command blits from bank 4, a separate from-ROM render still to be pulled (tracked in the ROADMAP) for both the 17- and 50-fief layouts.
