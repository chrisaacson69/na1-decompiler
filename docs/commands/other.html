Other

The system menu — sound, animation and message-speed toggles, plus Save and the end-turn / quit options. Settings, not strategy.
① decompiler → C ② data-walk labels ⚡ instant — no animation

What it does

Other (driver_other $B23E) opens a seven-entry system menu (Change which?). Each pick is dispatched through a jump table (jumptab_b9e8) to its own handler, and the menu loops until a handler signals exit or you choose End. The labels come straight from driver_other_data_ff1a. Most entries just flip a preference; Save writes the battery-backed game; and only End (option index 5) returns true to actually conclude the turn.

The seven options

Read from the label table $FF1A and the dispatch table jumptab_b9e8 ($B9E8).

#OptionWhat it does
1Soundsound on/off → audio_wait_gate (off silences via syscall_audio_control)
2Animat(ion)command & battle animations on/off → bit 2 of ai_turn_flags
3Waitmessage/text-advance speed → delay_loop_count = 2·n² (n = 1–10)
4Savemarks sram_save_pending_flag → the game saves at season end
5Battlewatch other daimyo’s battles play out → ui_confirm_flag_6e7d (read at $9130 to decide whether AI-vs-AI combat is shown)
6Endconclude the turn (index 5 — the only option that returns true)
7Menuback out to the command menu (index 6)

These are the same preferences the engine reads elsewhere — the Animat bit in ai_turn_flags gates every ui_helper_e80c command animation the other pages show; Wait’s delay_loop_count times every text box; Battle’s ui_confirm_flag_6e7d decides whether you spectate AI wars; and Save routes through the checksummed SRAM writer. IDing these is the grounding for the code that consumes them.

A settings command that quietly controls the rest of the game. Nothing here touches a province or a daimyo, but two entries matter: Save is the only persistence path, and the Animat toggle is the master switch for the command-animation VM every other page on this site renders. The seven handlers behind jumptab_b9e8 are small, self-contained UI routines — a settings panel, not a subsystem.
