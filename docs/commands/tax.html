Lord Command · No. 3

Tax

Set a province’s tax rate. Raising it drains loyalty and wealth (and costs charisma); lowering it restores them. Symmetric, one-shot — and scales with the CHANGE.

In game

The Tax animation, rendered purely from ROM bytes — no emulator capture.

Peasants sit amid the koku they owe — then the coins are taken and discontent sets in (the tax-raised animation, id 19). FROM ROM

When to use it

Set tax high — ~55% is the sweet spot — on turn one and leave it. Every re-fiddle re-charges the loyalty/wealth penalty, so a province whose rate you keep flipping bleeds far more than its harvest gains. Pushing past ~55–60% also opens Riot vulnerability (see the event system).

Changes: loyalty ±pct · wealth ±pct (same %, opposite the tax move) · province tax := new rate · daimyo charisma ∓1.
Under the hood — the verified bytecode

How it works

Driver $999A reads as a step-by-step script:

  1. $999FSelect Tax and the province; the window shows the current rate.
  2. $99BCPrompt: "Tax is N. Enter new tax." (capped at 100, or 30 under tax_helper_db12).
  3. $99DBNumber entry → the new rate. Cancel or no-change → bail.
  4. $99EFDirection branch. If raised → animation 19 + "The peasants are protesting." If lowered → animation 18 + "The peasants are delighted."
  5. $9A1CDaimyo charisma: −1 if you raised tax, +1 if you lowered it.
  6. $9A3BThe province’s tax byte is set to the new rate.
  7. $9A42loyalty ±= pct_op(loyalty, |Δtax|) via effect_tax ($82D6) — drops on a raise, recovers on a cut.
  8. $9A4Cwealth ±= pct_op(wealth, |Δtax|) — same magnitude. → confirm.

The exact formula

Read straight from $82D6 bytecode and execution-verified.

// Tax is symmetric: the hit scales with the CHANGE, not the level
Δ      = new_tax − old_tax            // signed

// effect_tax ($82D6) applied to BOTH loyalty and wealth:
stat_adjust(stat):
    move = pct_op(stat, |Δ|)        // ⌊stat × |Δ| ÷ 100⌋
    stat = (Δ > 0) ? stat − move    // raise tax → people sour
                  : stat + move    // lower tax → people warm

loyalty  = stat_adjust(loyalty)        // record +12
wealth   = stat_adjust(wealth)         // record +14
charisma += (Δ > 0) ? −1 : +1        // daimyo record +4
tax[fief] = new_tax
Symmetric and one-shot — the key strategic property. Raising the rate by |Δ| points immediately drains both loyalty and wealth by |Δ|% (and costs the daimyo 1 charisma); lowering it restores the same percentage (and earns 1 charisma). Because the penalty scales with the change and not the level, the optimal play is to set tax high once, early, and never touch it again — every adjustment is a fresh tax on your own people. Note the underlying effect_tax ($82D6) is a generic signed-percent primitive; the driver $999A calls it twice (loyalty, then wealth) with the tax delta.

Validated against the ROM

A tax change of |Δ| points moves BOTH loyalty and wealth by pct_op(stat, |Δ|), opposite the tax direction. Worked from the unit-verified pct_op primitive.
