Ninja (Hire ▸ Ninja)

Buy a ninja and send him on a sabotage mission against an enemy fief — uprising, revolt, flood, assassination or arson. The covert half of the Hire command.

What it does

Reached from the Hire command’s (Men/Ninja)? prompt — pick Ninja and you leave the recruiting sandbox for an offensive espionage subsystem (effect_ninja_sabotage $A2D2). You pay gold to hire the ninja (rate hire_gold_rate $6E13, which re-rolls every turn), choose a target fief (Send where?), then choose one of five missions. A Luck attempt-gate gates every mission: it proceeds only when your daimyo’s Luck stat (record +3) + 30 beats the target daimyo’s Luck — otherwise Your ninja failed! and you forfeit the gold anyway (effect_ninja_failed $A29B). On success the target loses the mission’s fields and you see Fief X’s <stat> has declined by N. This is the system the old notes mis-filed under “Bribe” — see the correction below; the genuinely separate Bribe command gets its own page.

How it works

The driver reads as a step-by-step script:

  1. $A615From Hire, answer (Men/Ninja)? with Ninja (ui_helper_d351) → effect_ninja_sabotage $A2D2.
  2. $A2EDPrompt "How many" — number entry (gated by affordability vs hire_gold_rate; no gold → "You have no gold.").
  3. $A31BPrompt "Send where?" — pick the target province.
  4. $A335Prompt "What mission" → the menu "1-Uprisng 2-Revolt 3-Dams 4-Assassin 5-Arson" ($BD8F).
  5. $A36DThe ninja sets out — ui_helper_e80c(12) plays the mission-launch animation (each mission then has its own effect animation; see the table).
  6. $A396Luck attempt-gate (all 5 missions). Proceeds only if target_daimyo.Luck < your_daimyo.Luck + 30 (record +3 = Luck) — a fixed +30 attacker edge.
  7. $A39AMission switch — dispatch to the chosen sabotage effect (drains via hire_stat_drain_rng, clamped so a field never goes below 0).
  8. $A2ADOn failure → "Your ninja failed!"; gold is spent regardless (effect_ninja_failed).

Drain magnitude (from hire_stat_drain_rng $A255 bytecode, confirmed 2026-06-11): drain = (rng % max(1, ⌊field × √(unit_count) ÷ 100⌋) + 1) × 5 — always a multiple of 5, scaling with the target field’s value and √(unit_count) (the ninjas you hired). Economics: cost ∝ unit_count (linear) but drain ∝ √unit_count → strong diminishing returns; it is pure denial (you gain nothing), best as small-batch pre-war softening. Each drain is clamped to the field (can zero a stat, never negative). Emulator-confirmed shape across the full RNG band (tools/probe-espionage.py — drains matched {5,10,…,X×5}). ⚠ Earlier text put the √ over your_skill+30 — wrong: it is √(unit_count); the +30 is the separate Luck attempt-gate. *Assassin has no field-drain animation — it runs the resolution sub (animation 1).

This is the “buy → ninja” system — and it is NOT Bribe. The toml long labeled $A2D2 effect_hire, and an earlier note filed the whole Uprising/Revolt/Dams/Assassin/Arson menu under the Bribe command. The bytecode says otherwise: this menu hangs off Hire ▸ Ninja (driver_hire $A5F4), while the recruit-men path is effect_hire_men $A553 and the real Bribe command ($8D4D) is a separate gold-for-spy peasant-defection. The label is now effect_ninja_sabotage. The sabotage mechanics the old note described were all real — only the command attribution was wrong.
